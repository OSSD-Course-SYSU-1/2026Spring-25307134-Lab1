# SimpleVideo 新增功能描述 — 倍速播放

## 一、新增功能概述

在原有视频播放器基础上，新增**多档倍速播放**功能，支持 1.0x / 1.25x / 1.75x / 2.0x 四档播放速度切换，同时优化交互：支持点击视频区域暂停/恢复播放。

## 二、新增功能详情

### 2.1 倍速播放选择器

- **位置**：视频播放页顶部，居中显示
- **可选倍速**：1.0x（默认）、1.25x、1.75x、2.0x（当前 SDK 不支持 1.5x）
- **交互**：点击任意倍速标签即切换，当前选中项高亮（金色加粗），其余半透明
- **生效时机**：播放过程中实时切换，无需暂停

### 2.2 点击视频区域暂停/恢复

- 点击视频画面任意位置可切换播放/暂停状态
- 替代原有必须在控制条点击播放/暂停按钮的操作方式

## 四、技术实现要点

### 4.1 倍速 API

使用 Video 组件的 `currentProgressRate` 属性，配合 `PlaybackSpeed` 枚举：

```typescript
Video({
  src: this.source,
  currentProgressRate: this.curRate,  // PlaybackSpeed 枚举值
  controller: this.controller
})
```

| 用户看到的 | PlaybackSpeed 枚举值 |
|-----------|---------------------|
| 1.0x | `PlaybackSpeed.Speed_Forward_1_00_X` |
| 1.25x | `PlaybackSpeed.Speed_Forward_1_25_X` |
| 1.75x | `PlaybackSpeed.Speed_Forward_1_75_X` |
| 2.0x | `PlaybackSpeed.Speed_Forward_2_00_X` |

> 注：当前 SDK 5.0.5(17) 不支持 1.5x 倍速（`Speed_Forward_1_50_X` 从 SDK 22 开始支持）。

### 4.2 状态同步方案

使用 `@StorageLink('curRate')` 实现倍速状态在多个组件间的同步：

```
VideoPlayerPage（顶部倍速按钮）
    │
    ├── 点击倍速标签 → 更新 @StorageLink('curRate')
    │                              │
    └── VideoPlayer                │
        ├── @StorageLink('curRate') ← 自动同步
        └── Video 组件重渲染，currentProgressRate 生效
```

### 4.3 点击暂停/恢复

将 `onClick` 从 Video 组件提升到 Stack 层级，避免 Stack 内 zIndex 更高的播放图标/加载动画拦截点击事件。

## 五、与基础工程的差异

| 对比项 | 基础工程 | 新增功能后 |
|--------|---------|-----------|
| 倍速切换 | 不支持 | 支持 4 档倍速（1.0x~2.0x） |
| 倍速控制位置 | 无 | 播放页顶部居中 |
| 视频点击 | 无响应 | 点击切换播放/暂停 |
| 控制条 | 播放/暂停 + 进度条 + 时间 | 不变（倍速按钮已移至顶部） |
