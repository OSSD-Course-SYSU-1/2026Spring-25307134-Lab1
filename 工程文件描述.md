# SimpleVideo 鸿蒙视频播放器 — 工程文件描述

## 一、项目概述

SimpleVideo 是华为官方 Codelabs 提供的 HarmonyOS 示例项目，基于 **ArkTS + ArkUI** 框架实现一个简易视频播放器。项目支持海报轮播、视频列表浏览、本地及网络视频播放、播放进度控制等功能。

- **系统要求**：HarmonyOS 5.0.5 Release+
- **IDE**：DevEco Studio 6.0.2 Release+
- **SDK**：HarmonyOS 6.0.2 Release SDK+
- **设备**：华为手机（标准系统）

## 三、核心源文件详解

### 3.1 应用入口 — `EntryAbility.ets`

```typescript
// 路径：entry/src/main/ets/entryability/EntryAbility.ets
// 继承 UIAbility，HarmonyOS 应用的唯一入口
// onWindowStageCreate() 中加载首页：pages/VideoListPage
```

- **生命周期**：onCreate → onWindowStageCreate → onForeground → onBackground → onWindowStageDestroy → onDestroy
- **关键操作**：`windowStage.loadContent('pages/VideoListPage')` 加载首页
- **日志**：使用 `@kit.PerformanceAnalysisKit` 的 hilog 记录各生命周期节点

### 3.2 全局常量 — `CommonConstants.ets`

```typescript
// 路径：entry/src/main/ets/common/constants/CommonConstants.ets
// 集中管理项目中所有硬编码常量
```

| 常量分类 | 包含项 |
|---------|--------|
| **路由常量** | `SECOND_PAGE`（播放页路径）、`SOURCE`（参数 key） |
| **网络地址** | `NET` — 网络视频 URL（引用 string 资源） |
| **时间格式化** | `START_TIME`（"00:00"）、`SPLIT`（":"）、`ZERO_STR`、"0" |
| **布局百分比** | `STRING_PERCENT` 枚举（Slider 宽度、视频高度、Swiper 高度等） |
| **边距/字号** | `MARGIN_FONT_SIZE` 枚举（7 个档位：12/16/18/24/50/94/132） |
| **层级样式** | `STACK_STYLE` 枚举（zIndex、进度条参数等） |
| **视频类型** | `VIDEO_TYPE` 枚举（inner/net） |

### 3.3 页面层（pages/）

#### VideoListPage.ets — 视频列表首页

```typescript
@Entry             // 标记为应用入口页面
@Component         // 声明为 ArkUI 组件
struct VideoListPage
```

**组件树**：
```
Navigation（路由容器）
└── Column（纵向布局）
    ├── SwiperVideo（轮播图组件）
    └── List（视频列表）
        └── ForEach → ListItem → VideoModule（列表模块）
```

**职责**：
- 作为 Navigation 根容器，绑定 NavPathStack 管理页面路由
- 组合 SwiperVideo 和 VideoModule 两个子组件
- 不包含业务逻辑，仅负责布局组装

#### VideoPlayerPage.ets — 视频播放页

```typescript
@Component
export struct VideoPlayerPage
```

**组件树**：
```
NavDestination（路由目标容器）
├── Row（返回按钮 + 文字）
└── Stack（层叠布局）
    ├── Image（播放按钮图标，暂停时显示）
    ├── Progress（加载动画，加载中显示）
    └── VideoPlayer（视频播放器，zIndex=0 底层）
```

**职责**：
- 接收路由跳转携带的视频源参数（通过 `@StorageLink('source')`）
- 通过 `@Provide` 向子组件注入 `VideoPlayerModel` 实例
- 管理返回按钮点击 → `pathStack.pop()`
- 页面隐藏时自动暂停播放

**状态变量**：
| 变量 | 装饰器 | 说明 |
|------|--------|------|
| `videoPlayerModel` | `@Provide` | 向子组件提供播放器状态 |
| `source` | `@StorageLink('source')` | 接收跨页面传递的视频源 URL |
| `isPlay` / `isLoading` | 来自 Model | 控制播放按钮和加载动画的显隐 |

### 3.4 视图层（view/）

#### IndexSwiper.ets — 轮播图组件

```typescript
@Component
export struct SwiperVideo
```

- 使用 `Swiper` 组件 + `ForEach` 循环渲染 `SWIPER_VIDEOS` 数据源
- `autoPlay(true)` 自动轮播
- 点击轮播图 → 设置 `source = $rawfile('videoTest.mp4')` → 跳转播放页

**子组件 SwiperItem**：
- 通过 `@StorageLink('source')` 写入视频源
- `pushPathByName('VideoPlayerPage', undefined)` 执行路由跳转

#### IndexModule.ets — 视频列表模块

```typescript
@Component
export struct VideoModule
```

- 接收 `moduleName` 参数（"最近播放"/"为你推荐"）
- 使用 `List` 横向滚动 + `ForEach` 渲染 `HORIZONTAL_VIDEOS` 数据源
- 点击视频条目 → 设置 `source = NET`（网络地址）→ 跳转播放页

**子组件 HorizontalItem**：
- 显示视频封面图 + 视频名称
- 点击时触发路由跳转

#### VideoPlayer.ets — 视频播放器核心组件

```typescript
@Component
export struct VideoPlayer
```

- 封装 HarmonyOS `Video` 组件
- 绑定 `VideoController` 进行播放控制
- 监听 5 个视频事件：

| 事件 | 回调 | 作用 |
|------|------|------|
| `onUpdate` | 实时获取 `event.time` | 更新当前播放时间 |
| `onPrepared` | 获取 `event.duration` | 获取视频总时长 |
| `onFinish` | `finish()` | 播放结束重置状态 |
| `onError` | 弹 Toast | 提示"请检查网络" |
| `autoPlay(false)` | — | 禁止自动播放 |

- 内部包含 `VideoSlider` 进度控制条

#### VideoPlaySlider.ets — 播放控制条

```typescript
@Component
export struct VideoSlider
```

**布局**：横向排列
```
[播放/暂停图标] [当前时间] [进度条Slider] [总时长]
```

**功能**：
- 播放/暂停按钮（通过 `iconOnclick()` 切换）
- 可拖动进度条（`Slider` 组件，min=0, max=durationTime）
- 进度条样式随操作状态变化（拖动时变细，透明度变化）
- 实时显示当前时间和总时长（格式 `mm:ss`）

### 3.5 模型层（model/）

#### VideoPlayerModel.ets — 播放器状态模型

```typescript
export class VideoPlayerModel {
  currentTime: number = 0;        // 当前播放时间（ms）
  durationTime: number = 0;       // 视频总时长（ms）
  isPlay: boolean = false;        // 是否正在播放
  isOpacity: boolean = false;     // 控制条透明度状态
  durationStringTime: string;     // 总时长的字符串格式 "mm:ss"
  currentStringTime: string;      // 当前时间的字符串格式 "mm:ss"
  flag: boolean = false;          // 视频是否加载完毕
  isLoading: boolean = false;     // 是否正在加载中
  progressVal: number = 0;        // 加载进度（0-100）
  controller: VideoController;    // Video 控制器实例
}
```

#### VideoControll.ets — 播放控制逻辑（纯函数）

| 函数 | 触发时机 | 逻辑 |
|------|---------|------|
| `prepared()` | 视频 onPrepared | 计算总时长并格式化、设置 flag=true |
| `finish()` | 视频 onFinish | 重置 isPlay、isOpacity |
| `sliderOnchange()` | Slider onChange | 更新 currentTime、调用 setCurrentTime seek |
| `changeSliderTime()` | 时间更新 | 毫秒数 → "mm:ss" 格式转换 |
| `iconOnclick()` | 播放/暂停按钮点击 | 切换播放状态、处理首次加载动画 |

## 四、数据流分析

### 4.1 页面路由数据流

```
VideoListPage                      VideoPlayerPage
    │                                    │
    │  点击轮播图/视频条目                │
    ├── @StorageLink('source') 写入URL    │
    ├── pushPathByName() ──────────────→ │
    │                                    ├── @StorageLink('source') 读取URL
    │                                    └── 传递给 VideoPlayer.src
```

- 使用 `@StorageLink('source')` 在页面间共享视频源 URL
- `AppStorage` 作为全局状态容器，任意组件可读写
- 无需通过路由参数显式传递，解耦页面间依赖

### 4.2 播放器状态数据流

```
VideoPlayerPage
    │
    │ @Provide videoPlayerModel
    ├──────────────────────────────┐
    │                              │
    ▼                              ▼
VideoPlayer                   VideoSlider
    │                              │
    ├── 监听 Video 事件             ├── 读取 isPlay（切换图标）
    ├── 更新 model.currentTime     ├── 读取 currentTime/durationTime
    ├── 更新 model.durationTime    ├── 读取 isOpacity（样式切换）
    └── 更新 model.flag            └── 调用 iconOnclick/sliderOnchange
```

- `@Provide` / `@Consume` 实现跨层级状态共享
- `VideoPlayerPage` 作为 Provider，`VideoPlayer` 和 `VideoSlider` 作为 Consumer
- 控制逻辑集中在 `VideoControll.ets` 纯函数中，组件只负责 UI 渲染

## 六、关键技术点

### 6.1 路由方案
- **Navigation + NavPathStack**：新版路由方案，通过 `pushPathByName()` 跳转，`NavDestination` 接收
- **route_map.json5** 配置路由表，`@Builder` 函数作为页面构建入口

### 6.2 状态管理
| 装饰器 | 用途 | 作用域 |
|--------|------|--------|
| `@Entry` | 标记应用入口页面 | — |
| `@Component` | 声明自定义组件 | — |
| `@Provide/@Consume` | 跨组件层级传递 VideoPlayerModel | 父子组件树 |
| `@StorageLink('source')` | 跨页面共享视频源 URL | AppStorage 全局 |
| `@Observed` | 标记数据类可观察（SwiperVideoItem 等） | — |

### 6.3 ArkUI 核心组件使用
| 组件 | 用途 | 关键属性 |
|------|------|---------|
| `Swiper` | 轮播图 | `autoPlay(true)` 自动播放 |
| `List` + `ListItem` | 视频列表 | `listDirection(Axis.Horizontal)` 横向滚动 |
| `Navigation` + `NavDestination` | 页面路由 | `mode(NavigationMode.Stack)` 单栏栈模式 |
| `Video` + `VideoController` | 视频播放 | `controls(false)` 隐藏默认控件，自建 UI |
| `Slider` | 进度条 | `style(OutSet)`, 支持拖动 seek |
| `Stack` | 层叠布局 | 播放按钮/加载动画/视频的分层 |
| `ForEach` | 循环渲染 | 遍历 SWIPER_VIDEOS / HORIZONTAL_VIDEOS |

### 6.4 视频控制设计
- 视频准备完成（onPrepared）→ 获取总时长 → 格式化显示 → 设置 flag=true
- 用户点击播放 → 检查 flag：已加载则直接播放，未加载则显示 loading 动画 + 定时轮询
- 进度条拖动 → `SliderChangeMode.Begin/Moving/End` 三阶段差异化处理
- 播放完成（onFinish）→ 重置 isPlay 和 isOpacity
- 页面隐藏（onPageHide）→ 自动 `controller.pause()`

### 6.5 时间格式化
- 毫秒值 → "mm:ss" 字符串（`changeSliderTime()`）
- 分钟小于 10 补零（如 "03:45"）
- Video 组件 onUpdate 事件高频触发，实时刷新当前时间显示
